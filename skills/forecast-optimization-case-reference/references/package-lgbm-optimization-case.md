# 真实预测优化案例参考：稀疏销量 / LightGBM Tweedie

只在用户明确要求参考该案例，或当前证据与低频、零膨胀、T+N 销量/需求预测相似时读取。本文件是参考案例，不是通用规则库。使用时必须先有当前实验证据，并把案例模式改写成当前实验可执行的验证假设。

## 案例背景

历史案例为 T+2 销量预测，主模型使用 `LightGBM LGBMRegressor` + `tweedie`，预测粒度聚焦到实体-对象-日期级别。该案例的有效思路是先收敛预测口径，再用可获得的历史、窗口、节假日、热度、语义和分层校准逐步修正系统性偏差。

不要把案例中的固定业务对象、字段名、数据路径、日期窗口或 run profile 套到当前实验。当前实验未明确是 LightGBM 或 Tweedie 时，不要建议修改固定模型参数；可以抽象为“稀疏非负目标的分布假设需要复核”。

## 可迁移原则

- 先确认预测粒度、标签、预测提前量和评估口径，再谈优化。
- 历史目标类特征必须满足预测提前量边界；T+2 场景只能使用 `ds-2` 及更早信息。
- 优先做可解释特征，不盲目堆特征。
- 后处理不能替代模型能力；只有切片证据支持时才做保守补偿。
- 分层校准和后处理必须使用预测时可得字段，并添加不可售、窗口结束、规则剔除等 guard。
- 日总量接近但明细误差大时，优先考虑分布、分层和对象-实体交叉特征，而不是全局上调。

## 案例中曾有效的方向

### 高低峰历史均值

案例中新增高低峰日期标记和高/低峰历史均值后，valid bias、test bias、daily valid、daily test 均明显改善。可迁移为：如果当前实验存在特殊日期、峰谷日、活动日或周期性高低峰，并且这些日期在预测时可知，可以构造“目标日期类型 + 安全历史均值”。

建议形态：
- 字段来源：预测时可知的日期属性 + 满足提前量边界的历史目标。
- 构造：按对象、实体或区域聚合，使用 1/2/3/7 等窗口的安全历史均值。
- 风险：日期属性不可在线获得或历史目标未错位会导致泄漏。

### 节假日位置

案例中节假日 first/middle/last 位置特征对总量校准有帮助，尤其改善 daily test。可迁移为：当误差集中在连续节假日、活动期或特殊周期中时，补充周期内位置特征，而不只使用是否节假日。

建议形态：
- 构造：`is_period_first_day`、`is_period_middle_day`、`is_period_last_day`、`period_day_idx`、`period_span_days`。
- 验证：周期内不同位置的 signed error 和 WAPE 是否收敛。

### 生命周期、窗口和可售状态

案例中可售状态、售卖/使用窗口、剩余天数、对象年龄和新品 bucket 用于处理不可售正预测、新品冷启动和临近结束误差。可迁移为：对任何存在上下架、有效期、可用窗口或生命周期的预测对象，优先补充预测目标日视角的窗口特征。

建议形态：
- 构造：目标日是否可用、距离开始/结束天数、剩余天数、对象年龄、首次出现天数、新品 bucket。
- 验证：不可用样本正预测率、new_window/ending_window 切片误差。

### 热度、活跃度和分布

案例中补充对象在实体、城市/区域、全局和组合层级的热度、份额、活跃率、可售日归一化指标，用于改善低频和新品分布误差。可迁移为：当总量尚可但明细分配差，或低频/新品误差贡献大时，补充“对象在不同层级中的相对位置”。

建议形态：
- 构造：安全 lag 的实体-对象销量、对象全局销量、区域对象销量、对象活跃天数、可售日均、份额、短窗/长窗比值、热度分位。
- 验证：明细粒度误差、低频/新品切片、分组校准前后指标。

### 分层校准与切片后处理

案例中对稀疏切片做保守后处理，并用组合层级需求对明细预测按比例拉齐。可迁移为：只有当 valid 或历史评估证据显示某些预测时可识别切片存在稳定偏差时，才建议校准或后处理。

建议形态：
- 分组：实体、对象类别、组合、日期、可售状态、热度等预测时可得字段。
- 保护：不可用、临近结束、规则剔除、高热已高估等样本不做不安全上调。
- 验证：被调整样本数量、调整前后切片误差、整体 Bias 是否恶化。

- 写“案例显示类似方向曾有效”，不要写“当前实验一定应使用该方向”。
- 每条建议要落到可执行实验：字段来源、构造口径、代码接入点、验证指标和风险。
- 如果当前源码没有参数或特征函数接入点，建议先定位接入链路。
