## 2. 优化轨迹

### 2.1 指标变化

| 阶段 | WAPE | Bias | WAPE改善 | 决策 |

| **baseline** (原始) | 0.7009 | +0.1695 | — | — |

**最优方案: trial_025 — WAPE 从 0.7009 降至 0.6206，改善 11.5%，Bias 从 +0.1695 降至 +0.0868，改善 48.8%。**

### 2.2 优化内容

**trial_023** (第1次，tweedie + 全特征):
- 目标函数: regression → Tweedie (variance_power=1.2)
- 分配策略: full → recent weighted (half_life=7d)
- 启用: T+2可用性特征、节假日位置、套餐活动力、分组校准
- 源码修复: `numeric_column_or_default()` 替代 `pd.to_numeric().fillna()` (8处)

**trial_025** (第2次，在023基础上加强):
- 延续 trial_023 的全部 tweedie + 特征配置
- 进一步增强: 套餐活动力特征的粒度、生命周期分桶交互

### 2.3 回退链

```
trial_028 尝试在 025 基础上继续优化
  → 训练失败 (data路径问题)
    → codegen retry 1: 修复路径 → 训练成功但输出无split列
    → codegen retry 2: 修复输出格式 → 仍无split列
    → 终极回退: 原始脚本 baseline preset
    → ROLLBACK
```

## 4. 结论

2 次成功实验将 WAPE 从 **0.7009 降至 0.6206（-11.5%）**，Bias 从 **+0.1695 降至 +0.0868（-48.8%）**。总消耗 $11、4 小时。

trial_028 的链式执行（从 trial_025 继承代码）验证了 `--previous-trial` 机制可工作，但输出格式兼容性问题导致评测失败。后续需加强 codegen 对输出列格式的约束。
